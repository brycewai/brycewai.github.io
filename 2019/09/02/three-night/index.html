最近疯狂的三天

2019.09.02 Bryce Wei 个人纪录片  热度 ℃

忍不住必须要记录一下，最近技术上遇到了一个巨坑，让我在这个年纪还连续通宵了3天，必须得单独开一篇

目的

由于某些原因，需要逆向分析xxx APK，主要涉及到三大方法：

  • 抓包流量分析
  • 静态代码分析
  • 动态调试分析

前面两种涉及到的环境比较容易，最后一个动态调试分析，随着攻防技术的提升以及Android系统本身的升级，已经不那么容易了，这记录的三天就在攻陷最后一种动态调试分析的环境问题

开启动态调试的三种方案

  1. AndroidManifest.xml
    在该文件中的application中添加android:debuggable="true"，再编译、签名、安装，进行调试。但是这个方法对自己写的app很有效，对别人尤其是签名严格的应用来说，基本不可行了，重新编译签名后的app往往没办法正常运行。

  2. ro.debuggable
    具体方法可选内存注入、修改boot.img后重刷系统等.一个是每次都需要重新注入，一个需要重刷系统，可自行选择

  3. xposed框架下的debuggable模块
    既然已经安装好了强大的xposed框架，那理论上应该很容易的就能够解决这个问题。搜索了一下，目前比较新的是App Debuggable.

硬件环境

移动端：

  • Redmi Note 7
  • Android 9(API 28)

PC端：

  • Mac OS
  • Ubuntu 19
  • Windows 10

周四

持续到周五凌晨3点半，才睡觉

主要准备进行第二种方法的尝试，而在尝试的过程中，出现了刷机问题。

  1. 先尝试刷开发版，看能否直接打开Debuggable模式，于是从官方稳定版，刷到最新的官方开发板，没问题，但是没有解决Debuggable模式问题
  2. 尝试解压boot.img，但是发现很多工具无法在Android 9上正常运行，于是就有思路能否刷回到Android 8版本，开始寻找第三方ROM，尝试的心态去刷机，然而发现要从最新的官方开发版刷回稳定版或者低版本，必须线刷，更尴尬的是，启动到Recovery模式时发现，TWRP recovery已经被重置回官方recovery了，本来分分钟就能刷回来的事情，却耗费了好几个小时…
  3. 一般来说，刷回TWRP recovery只需要准备好手机，TWRP recovery包，然后连上手机，刷就好了。然后这些都没问题，但是在fastboot flash的时候，先能找到设备，几秒后就找不到了，或者先能找到设备，本来几秒钟就能刷完的，结果刷了十几分钟还没结束，各种没出现过的问题都来了。
  4. 时间就耗在这里，Google了一圈，也有遇到类似的，说是驱动问题，然而我adb完美，不可能是驱动，但是死马当活马医，都试了一遍，问题点还是在那里。开始也怀疑是不是数据线或者接口问题，但是，adb模式却没有问题，手机开机后的连接也没问题，并且当时手头只有一根数据线一台台式机，没法验证这个猜想，就先搁浅了。
  5. 后来事实证明，问题真的就在这里，手机关机进入fastboot模式后，电脑能短暂识别到fastboot模式的手机大约3-5秒，后面就绝对断。第二天到公司用Mac和Win10笔记本都没问题。

当时没法验证，就各种其他方法折腾，最后在凌晨3点的时候只能用排除法勉强定位到问题很大概率是在电脑USB口或者数据线问题，但是因为当时手头只有windows 10台式机，无法进行对比定位，所以得第二天去公司用不同的机子进行验证。

周五

持续到周六凌晨5点
周五白天挤出很多时间，实验了方法3，刷的Redmi Note 7官方稳定ROM、官方开发ROM、官改ROM好几个ROM下的各种xposed框架，终于有找到适合红米Note 7的，并且其他Xposed插件也能在该Xposed框架下运行，但是就各种Debuggable插件不行，能装的5个Debuggable插件全试过了，各种刷系统、各种刷框架、各种装插件、各种重启，就是死活不行…

于是，今天晚上决定换个方案，主攻方案2，修改boot.img

尝试了很多网上高分方案未果（这里就不一一列出来失败方案了，免得误导后来者）后，终于柳暗花明的看到了一种很新并且作者仍然在维护的方法：SuperR's Kitchen
找了很多资料后，发现这篇微博是讲的很详细的怎么去用这个玩意儿的博客了，赞一个
然而，在解包/打包boot.img那一步出了问题，最新的SuperR中这一项是标红的，没办法选择，不知道是因为我用的免费版的，还是因为不支持Android 9的原因。然而，一看正式版15刀，在不确定原因的情况去买，有点心疼，更何况最近汇率飚升…该方案也是暂告失败，看看时间，凌晨5点，身体刚不住了，洗洗睡了…

周六

持续到周日凌晨4点半，好在，终于搞定了这个问题
具体过程就不详细描述了，因为那时候人已经接近恍惚了，搞完就直接睡了，现在去回忆大脑本身就有点抗拒还有点记不得了。过程总是艰难的，好在结果还是令人愉悦的。下面给出最终的解决方案。

最终的解决方案：

  • 硬件不变，仍然是Redmi Note 7
  • 系统底层仍然是Android 9，刷了crDroid v5.7，点击详情
  • Recovery刷的TWRP-3.3.0
  • Magisk框架，及其Root
  • Xposed框架
    • Magisk Riru
    • ED Xposed
    • EdXposed Installer
  • App Debuggable

The End
